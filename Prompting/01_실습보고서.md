### 생성 파일

| 경로 | 내용 |
|------|------|
| `Report/실습보고서.md` | README 5단계 회고용 본문 |
| `Report/README.md` | Report 디렉터리 안내 |

### `실습보고서.md` 구성

1. **실습 목표·달성도** — Activities 체크리스트(현재: 분석·`.cursorrules` 완료, 리팩터·UT·기능은 미완)
2. **도메인·아키텍처** — BMI·분류·보정 규칙, 모듈 구조
3. **Before & After** — 실제 코드 기준 스멜 표(Long Method, Magic Number, `getBmiRatio` 중복 등) + 목표 상태
4. **생성형 AI 활용** — `.cursorrules` 작성, 장단점, 다음 단계 프롬프트 예시
5. **단위 테스트 전략** — Given-When-Then / `TEST_F`, 우선순위, 작성 팁
6. **클린코드 회고** — 예상 장점·어려움
7. **다음 작업 계획** — 리팩토링 → UT → 기능 순

현재 저장소 상태(테스트 `FAIL()`, 본격 리팩터 미착수)를 반영해 **솔직한 진행률**로 적어 두었습니다. 2~4단계를 마친 뒤 §1·§3 체크리스트만 갱신하면 됩니다.

추가로 원하시면 `Report/fixtures/` 샘플 CSV나 발표용 1페이지 요약도 만들 수 있습니다.
