## 요약

`SHealth` 단일 클래스에 I/O·파싱·보정·BMI·통계가 집중된 **God Method**(`calculateBmi`, L6–108)가 핵심 리스크이다. **SRP·OCP·DIP** 위반이 두드러지고, **LSP**는 해당 없다. **P0** 이슈 8건: God Method, BMI=25 미분류, 0 나눗셈, 중복 분류, FAIL 스텁 등. 2단계 리팩토링은 `.cursorrules` 순서(네이밍 → 상수화 → 추출 → SRP 준비)와 TC 16·7·20 선행을 따른다.

## 2. Code Smell (Fowler + P0/P1/P2)

| 우선순위 | 건수 | 대표 스멜 |
|:--------:|:----:|-----------|
| **P0** | 8 | Long Method, Duplicated Code, Division by Zero, BMI=25 버그, Ignored Test |
| **P1** | 9 | Magic Number, Primitive Obsession, Array Overflow, height=0 |
| **P2** | 5 | Data Clumps(24변수), Inappropriate Intimacy(I/O) |

### 2.1 P0 상세

| 문제점 | 스멜 | 영향 | 개선 | TC |
|--------|------|------|------|:--:|
| `calculateBmi` ~100줄 | Long Method | 테스트·가독성 | private 추출 | 1–4, 32 |
| 분류 `>25` vs README `≥25` | Bug | 25.0 미분류 | `>= 25.0` | **16** |
| `sum/ageCount`, `*100/sum` | 0 나눗셈 | NaN/크래시 | 가드 | **7**, **20** |
| 연령대·분류 중복 if | Duplicated Code | 버그 복제 | `classifyBmi` 단일화 | 11–18 |
| `FAIL()` 스텁 | Ignored Test | Green 불가 | 제거 | **37** |
